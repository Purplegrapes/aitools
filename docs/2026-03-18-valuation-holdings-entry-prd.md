# 宝倍估值 持仓同步 PRD

## 1. 背景

当前宝倍估值已经具备“我的持仓”查看能力，但持仓新增链路仍然存在两个问题：

- 入口分散，用户要在“添加持仓”“上传持仓”等不同入口之间理解差异
- 上传截图链路不够直接，用户更自然的预期是“选图后直接识别，再确认导入”

因此，需要把持仓新增能力收口为“同步持仓”单入口，再分流到手动录入或上传截图两条路径。

## 2. 产品目标

- 让用户用最短路径完成持仓录入
- 保留手动录入能力，适配少量持仓场景
- 支持上传持仓截图自动识别，降低批量录入成本
- 自动识别结果必须先确认，再导入正式持仓

非本期目标：

- 不做券商/API 直连同步
- 不做自动持续同步
- 不做 Excel / CSV 导入
- 不做失败项逐条重匹配

## 3. 目标用户

- 已持有 1 到 10 只基金，希望快速录入并查看收益的普通用户
- 不愿逐条输入，但愿意上传截图辅助识别的用户
- 识别失败后愿意重新上传或切回手动录入的用户

## 4. 一句话需求定义

“我的持仓”提供一个统一的“同步持仓”入口。用户先选择手动录入或上传截图；上传截图后直接唤起选图，系统只保留识别成功的基金进入确认页，用户确认后再导入。

## 5. 信息架构

1. 我的持仓页
2. 同步持仓页
3. 手动录入页
4. 持仓确认页
5. 修改持仓页

## 6. 页面与流程

### 6.1 我的持仓页

#### 页面目标

展示组合收益，并提供唯一的持仓新增入口。

#### 页面要求

- 页面标题保留“我的持仓”
- 有持仓时，页头只保留一个“同步持仓”按钮
- 空状态下也只保留一个“同步持仓”按钮
- 不再展示底部操作栏

### 6.2 同步持仓页

#### 页面目标

作为持仓录入的唯一入口，帮助用户先选择同步方式。

#### 页面结构

- 手动录入
- 上传截图

#### 交互要求

- 点击“手动录入”进入手动录入页
- 点击“上传截图”直接唤起图片选择
- 选图并识别成功后，直接进入持仓确认页
- 如果识别为空或失败，提供重新上传和转手动录入

### 6.3 手动录入页

#### 页面目标

适合录入少量持仓，快速完成保存。

#### 用户操作

1. 搜索基金名称或代码
2. 选择基金
3. 输入当前持有金额
4. 输入当前持有收益
5. 点击保存

#### 页面字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 基金名称/代码 | 是 | 支持搜索选择 |
| 持有金额 | 是 | 当前该基金总持有金额 |
| 持有收益 | 是 | 当前累计收益，亏损可填负数 |

#### 系统自动换算

- 持有份额 = 持有金额 / 当前净值
- 成本金额 = 持有金额 - 持有收益
- 成本净值 = 成本金额 / 持有份额

#### 录入口径说明

- 持有金额填“你当前这只基金值多少钱”
- 持有收益填“当前累计赚了/亏了多少钱”
- 系统会自动换算成持仓份额和成本净值

### 6.4 上传截图识别

#### 页面目标

让用户通过上传截图，减少重复输入。

#### 用户操作流程

1. 在“同步持仓”页选择上传截图
2. 立即唤起图片选择
3. 系统执行 OCR 识别
4. 系统只保留识别成功的基金进入确认页
5. 用户确认金额和收益后保存

#### 支持的输入

- 单张截图
- 多张截图

### 6.5 持仓确认页

#### 页面目标

把“自动识别”变成“自动识别 + 人工确认”，降低错误入库风险。

#### 页面结构

- 顶部摘要：已识别出多少只基金，请确认后保存
- 识别结果表格
- 底部操作：重新上传、确认导入

#### 表格字段

| 字段 | 是否允许编辑 | 说明 |
| --- | --- | --- |
| 基金 | 否 | 展示基金名称与代码，名称单行截断 |
| 持有金额 | 是 | 支持编辑 |
| 持有收益 | 是 | 支持编辑 |
| 操作 | 否 | 仅支持删除 |

#### 保存规则

- 仅识别成功项进入确认页
- 用户可删除不想导入的记录
- 其余记录批量确认导入

## 7. 功能规则

### 7.1 基金匹配规则

- 优先通过基金代码匹配
- 若无代码，则使用基金名称模糊匹配
- 若未匹配成功，不进入确认页

### 7.2 金额与收益识别规则

- 允许识别字符串中带千分位与货币符号
- 允许收益为负数
- 若数值格式异常，不进入确认页

### 7.3 最终入库口径

无论是手动录入还是截图识别，最终入库字段应统一为：

- 基金代码
- 基金名称
- 持有份额
- 成本净值

## 8. 异常场景与处理策略

### 8.1 截图中未识别出基金

- 提示“暂未识别出持仓信息”
- 提供“重新上传”和“转手动录入”

### 8.2 识别出基金名，但匹配不到基金

- 该条不进入确认页
- 记录失败原因，供日志或提示使用

### 8.3 金额或收益识别异常

- 该条不进入确认页
- 建议用户重新上传或改走手动录入

### 8.4 部分成功、部分失败

- 只展示成功项
- 失败项不阻塞导入

## 9. 交互与体验原则

### 9.1 入口统一

- 用户只需要理解一个入口：“同步持仓”

### 9.2 自动识别必须可复核

- OCR 结果不能直接入库
- 必须经过用户确认

### 9.3 路径直接

- 上传截图时直接唤起选图
- 不增加额外中转页面

### 9.4 确认页足够轻

- 使用表格而不是卡片
- 只保留识别成功项
- 只允许必要编辑

## 10. 文案建议

### 10.1 同步持仓页

- 标题：`同步持仓`

### 10.2 手动录入页

- 标题：`添加持仓`

### 10.3 上传截图动作

- 按钮：`上传截图`
- 说明：`适合一次录入多只基金，系统会先识别，再由你确认。`

### 10.4 识别结果确认页

- 标题：`确认持仓信息`
- 说明：`识别结果可能有误，保存前请确认金额和收益。`

## 11. 数据与接口建议

### 11.1 手动录入接口

沿用当前持仓保存接口，只要能接收统一的最终字段即可。

### 11.2 截图识别接口

#### 识别接口

输入：

- 图片文件

输出：

- 识别出的基金列表
- 每条记录的识别状态

#### 确认导入接口

输入：

- 用户确认后的持仓列表

输出：

- 导入成功条数

## 12. MVP 范围

### P1

- 手动录入流程收口
- 同步持仓入口
- OCR 识别结果确认页
- 成功/失败状态闭环

### P2

- 多张截图合并识别
- 基金自动匹配优化
- 识别历史缓存

### P3

- Excel / CSV 导入
- OCR 识别置信度提示

## 13. 验收标准

### 功能验收

- 用户可以通过手动录入成功新增一只基金持仓
- 用户可以上传持仓截图并看到确认列表
- 用户可以调整金额和收益后保存
- 识别失败时，不影响成功项导入

### 体验验收

- 用户能在 10 秒内理解“同步持仓”页的两种方式
- 用户能在识别失败时找到明确的下一步动作
- 导入后能顺畅回到“我的持仓”查看结果

## 14. 结论

“同步持仓”应成为持仓录入的唯一入口，而不是继续保留多个分散入口。

用户要做的只有一件事，就是把自己的基金持仓同步进宝倍估值。

实现上提供两条路径：

- 手动录入
- 上传持仓截图自动识别

其中，截图识别不是直接导入，而是“选图 + 识别 + 确认 + 保存”的闭环，这会是这条能力真正可用的关键。
